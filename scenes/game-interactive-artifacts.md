## English

### Business Core Abstraction

Turn prompts or creative descriptions into playable games, interactive demos, HTML5 games, Pygame projects, or retro-style game artifacts.

### What To Compare

| Axis | Questions / Difference |
| --- | --- |
| Production object | Browser game, HTML5 game, Pygame game, interactive demo, or retro game? |
| Operating space | Prompt pipeline, live preview, game editor, or multi-agent pipeline? |
| Human control | Can the user edit mechanics, assets, state, or code? |
| Delivery boundary | Playable artifact, source package, or hosted game? |
| Risk | Is it a demo, or a repeatable production workspace? |

## 中文

### 业务核心抽象

把 prompt 或创意描述变成可玩的游戏、互动 demo、HTML5 game、Pygame project 或复古游戏产物。

### 这一章要比较什么

| 维度 | 问题 / 差异 |
| --- | --- |
| 生产对象 | browser game、HTML5 game、Pygame game、interactive demo 还是 retro game？ |
| 操作空间 | prompt pipeline、live preview、game editor 还是 multi-agent pipeline？ |
| 人类控制 | 用户能否编辑机制、素材、状态或代码？ |
| 交付边界 | playable artifact、source package 还是 hosted game？ |
| 风险 | 它是 demo，还是可重复生产 workspace？ |
