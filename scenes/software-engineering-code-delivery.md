# Software Engineering / Code Delivery

## Business Core Abstraction

Turn issues, bugs, product requirements, repo context, and engineering tasks into code changes, test results, commits, runnable projects, or pull requests.

## What To Compare

| Axis | Questions |
| --- | --- |
| Task boundary | Is this small edit, long task, issue-to-PR, or multi-agent session management? |
| Operating space | IDE, terminal, browser workspace, agent session board, sandbox, cloud workspace? |
| Human control | Diff approval, PR review, session takeover, test execution, tool constraints? |
| State and memory | Repo context, traces, sessions, tests, PR history, task plans? |
| Delivery boundary | Diff, commit, PR, deployed project, or completed coding task? |
