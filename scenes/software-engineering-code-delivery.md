## 中文

### 业务核心抽象

把 issue、bug、产品需求、repo 上下文和工程任务，变成代码改动、测试结果、commit、可运行项目或 PR。

### 这一章要比较什么

| 维度 | 问题 / 差异 |
| --- | --- |
| 任务边界 | 小代码编辑、长任务、issue-to-PR，还是 multi-agent session 管理？ |
| 操作空间 | IDE、terminal、browser workspace、agent session board、sandbox 还是 cloud workspace？ |
| 人类控制 | diff approval、PR review、session takeover、测试执行还是工具约束？ |
| 状态与记忆 | repo context、trace、session、test、PR history 还是 task plan？ |
| 交付边界 | diff、commit、PR、deployed project 还是 completed coding task？ |
