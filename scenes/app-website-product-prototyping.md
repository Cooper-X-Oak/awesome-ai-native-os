# App / Website / Product Prototyping

## Business Core Abstraction

Turn a product idea, prompt, existing website, or product requirement into a runnable app, website, or prototype that can be previewed, edited, shipped, or continued by humans.

## What To Compare

| Axis | Questions |
| --- | --- |
| Input object | Is the input a prompt, website, product brief, design, or repo? |
| Production object | Is the output a UI fragment, full app, mobile app, or deployed website? |
| Operating space | Is work done in a local workspace, browser runtime, IDE, or chat-preview loop? |
| Human control | Can the user edit code, edit visually, inspect preview, run tests, or deploy? |
| Delivery boundary | Does the project deliver code, hosted site, Expo app, or deployable project? |
