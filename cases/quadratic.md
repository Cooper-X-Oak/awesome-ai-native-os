## English

### Case Status

standard

### Evidence Level

E3 strong

### One-Line Positioning

Quadratic turns the spreadsheet grid into an AI-native data workspace where humans, AI, formulas, Python, SQL, JavaScript, charts, and external data sources operate on the same visible sheet.

### Business Scene

Data analytics / BI / financial analysis, especially exploratory analysis, spreadsheet modeling, dashboards, and analyst workflows.

### Core Business Object

A spreadsheet file that combines cells, data tables, formulas, code, charts, AI prompts, and connected data sources.

The core object is not a static spreadsheet. It is a programmable grid where each cell region can hold data, code execution, visual output, and analytical context.

### Production Artifact

Spreadsheet analysis, charts, dashboards, models, data transformations, and shareable analytical workbooks.

### Operating Space

Spreadsheet grid with integrated code and AI.

Quadratic's public materials position it as a modern spreadsheet with Python, SQL, formulas, JavaScript, charts, AI assistance, and database/API connections in one canvas.

### AI Operation Model

AI acts as an analyst and coding assistant inside the grid.

It helps write formulas, Python, SQL, and JavaScript; explains data; generates charts; transforms data; and supports analytical workflows while keeping the results visible in the spreadsheet.

### Human Control Model

Human control remains spreadsheet-native.

The user edits cells, inspects formulas and code, sees outputs in the grid, adjusts charts, validates data, and keeps direct control over the analysis structure.

### State And Memory

Quadratic preserves sheet state, formulas, code cells, query results, charts, external connections, cell-level outputs, and AI-assisted analysis context.

### Delivery Boundary

The boundary is a spreadsheet/workbook that contains the analysis, data, code, charts, and shareable outputs.

### Same-Scene Contrast

Quadratic differs from Clay by centering analytical computation rather than GTM enrichment workflows.

Quadratic is spreadsheet/code-first: the user computes, models, queries, and visualizes data. Clay is table/workflow-first: the user enriches, scores, researches, and activates leads/accounts.

### Reusable Pattern

Use the grid as a shared operating system for humans, AI, code, and data.

The reusable structure is:

1. grid as durable state,
2. cells as visible control points,
3. formulas/code as executable logic,
4. external data connectors,
5. charts as live artifacts,
6. AI as analyst/code helper,
7. workbook as delivery package.

### Reusable Recipe

1. Keep data, code, and visual outputs in the same spatial workspace.
2. Let each analysis step leave visible state in the grid.
3. Make AI write inspectable formulas/code rather than opaque answers only.
4. Preserve external data connections as part of the workbook.
5. Let users validate results by inspecting cells and code.
6. Treat the workbook as both workspace and artifact.

### Inferred Pattern

- Quadratic's durable abstraction is the spreadsheet as a programmable operating surface.
- Its OS lesson is that AI data work is easier to trust when every step lands in cells, code, or charts the human can inspect.
- Its tradeoff is that spreadsheet flexibility can become governance complexity for teams unless data lineage and permissions are explicit.

### Open Questions

- How granular is provenance across AI-generated code, external data, and cell outputs?
- How well does the workbook model scale for large production BI workflows?
- Which collaboration and permission primitives matter when AI can write code and query data in a shared sheet?

### 推断出的模式

- Quadratic 的稳定抽象是 spreadsheet as programmable operating surface。
- 它的 OS 启发是：每一步 AI data work 都落到 humans 可检查的 cells、code 或 charts 时，更容易被信任。
- 它的 tradeoff 是：spreadsheet flexibility 对团队来说可能变成 governance complexity，除非 data lineage 和 permissions 足够显式。
