# Contributing

> Contributions should improve the atlas as a decision tool, not only add links.
>
> 贡献应该提升这个图谱作为判断工具的价值，而不是只增加链接。

## Language

- [English](#english)
- [中文](#中文)

---

## English

### Core Rule

This repository is bilingual by default.

Before contributing public-facing documentation, read [docs/bilingual-structure.md](docs/bilingual-structure.md).

Do not submit an English-only public guide and assume it can be translated later. The bilingual structure is part of the baseline.

### Good Contributions

Useful contributions usually do one of these:

```text
add a missing AI-native production system
improve a scene comparison
complete a case card
add evidence for a candidate
downgrade a weak project with a clear reason
clarify a reusable pattern
improve bilingual documentation consistency
```

### Add A Project

When adding a project, include:

```text
name
official URL
business scene
core business object
production artifact
operating space
AI operation model
human control model
state and memory
delivery boundary
evidence links
grade suggestion
```

### Writing Style

Be concrete.

Prefer:

```text
This system turns a lead/account table into an AI-operable GTM workspace.
```

Avoid:

```text
This is an amazing AI tool for productivity.
```

### Evidence Discipline

Separate evidence from interpretation:

```text
Evidence: the project describes itself as a node-based AI studio for images, video, audio, and text.
Interpretation: this makes it a node-canvas-first multimodal production OS candidate.
```

### Documentation Checklist

For public documentation changes:

```text
Use the Language / English / 中文 structure.
Keep English and Chinese sections complete.
Do not use inline translation-label blocks inside an English document.
Keep links consistent across both language sections.
Use the glossary in docs/bilingual-structure.md.
```

## 中文

### 核心规则

本仓库默认采用中英双语结构。

贡献公开文档前，请先阅读 [docs/bilingual-structure.md](docs/bilingual-structure.md)。

不要提交英文单语公开说明文档，然后假设以后再补中文。双语结构是基本要求。

### 好的贡献

有用的贡献通常包括：

```text
添加遗漏的 AI-native production system
改进某个业务场景的比较方式
补完整案例卡
为候选项目补证据
用明确理由降级弱候选
澄清可复用模式
改进双语文档一致性
```

### 新增项目

新增项目时，请包含：

```text
名称
官方 URL
业务场景
核心业务对象
生产产物
操作空间
AI 操作模型
人类控制模型
状态与记忆
交付边界
证据链接
建议等级
```

### 写作风格

写具体。

推荐：

```text
This system turns a lead/account table into an AI-operable GTM workspace.
```

避免：

```text
This is an amazing AI tool for productivity.
```

### 证据纪律

把证据和解释分开：

```text
Evidence: the project describes itself as a node-based AI studio for images, video, audio, and text.
Interpretation: this makes it a node-canvas-first multimodal production OS candidate.
```

### 文档检查清单

公开文档修改需要检查：

```text
使用 Language / English / 中文 结构。
英文和中文章节都要完整。
不要在英文文档中插入夹杂式翻译标签块。
两种语言章节里的链接要一致。
使用 docs/bilingual-structure.md 中的固定术语。
```
